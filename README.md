# Portfolio Risk Analysis — Snowflake Streamlit + Notebook

> **⚠️ DISCLAIMER:** This code was generated by [Cortex Code](https://docs.snowflake.com/en/user-guide/cortex-code/cortex-code) and is intended for **testing, demonstration, and non-production use only**. It is provided as-is without warranty. Review and harden before using in any production environment.

Interactive loan portfolio risk analysis dashboard that queries Snowflake data in real time, builds visualizations, trains a simple ML risk model, generates an AI executive summary via Snowflake Cortex, and exports everything to a native-chart PowerPoint deck.

Two versions are provided and kept in sync:
- **Streamlit app** (`streamlit_app.py`) — deploy to Snowflake or run locally
- **Snowflake Notebook** (`loan_portfolio_analysis.ipynb`) — run in Snowsight or locally

---

## Prerequisites

- Snowflake account with access to Cortex AI (`SNOWFLAKE.CORTEX_USER` database role)
- A warehouse
- Python 3.11+ (for local development)
- Snowflake CLI (`snow`) for deployment
- A named connection in `~/.snowflake/connections.toml`

## Quick Start

### 1. Create your config

Copy the example and fill in your values:

```bash
cp config.example.py config.py
```

Edit `config.py`:

```python
CONNECTION_NAME = "default"                 # Your ~/.snowflake/connections.toml section name
DATABASE = "MY_DATABASE"                    # Your database
SCHEMA = "MY_SCHEMA"                        # Your schema
WAREHOUSE = "MY_WAREHOUSE"                  # Your warehouse
ROLE = "MY_ROLE"                            # Your role
STAGE = "@MY_DATABASE.PUBLIC.MY_STAGE"      # Stage for PPTX uploads
CORTEX_MODEL = "claude-sonnet-4-6"          # Cortex LLM model
APP_TITLE = "Portfolio Risk Analysis"       # Display title
```

> `config.py` is git-ignored so your credentials and environment details stay local.

### 2. Create test data

Run `setup_mock_data.sql` in Snowflake to create a test database with deterministic sample data (66 rows across 3 tables + a stage):

```bash
snow sql -f setup_mock_data.sql
```

Or paste the contents into a Snowflake worksheet. Then update `config.py` to point at the database you created (default: `PPTX_TEST`).

### 3. Run the Streamlit app

**Deploy to Snowflake (Streamlit in Snowflake):**
```bash
snow streamlit deploy
```

No additional connection config is needed — `st.connection("snowflake")` auto-connects in SiS.

**Run locally:**
```bash
pip install streamlit plotly pandas python-pptx lxml scikit-learn snowflake-connector-python snowflake-snowpark-python
streamlit run streamlit_app.py
```

For local Streamlit, `st.connection("snowflake")` reads from `~/.snowflake/connections.toml` (the `[default]` section). If your connection has a different name, create `.streamlit/secrets.toml`:

```toml
# .streamlit/secrets.toml  (git-ignored)
[connections.snowflake]
connection_name = "your_connection_name"
```

### 4. Run the notebook

**In Snowflake (Snowsight):**
Upload `loan_portfolio_analysis.ipynb` and `config.py` to a Snowflake Notebook. The notebook auto-detects the container runtime via `get_active_session()`.

**Locally (Jupyter / VS Code):**
```bash
pip install snowflake-snowpark-python python-pptx lxml scikit-learn plotly pandas
jupyter lab loan_portfolio_analysis.ipynb
```

The notebook reads `CONNECTION_NAME` from `config.py` and falls back to `Session.builder` with your `~/.snowflake/connections.toml` named connection. Override with:
```bash
SNOWFLAKE_CONNECTION_NAME=your_connection_name jupyter lab loan_portfolio_analysis.ipynb
```

## What It Does

1. **Portfolio KPIs** — Total loans, funded amounts, credit scores, delinquency rates
2. **Delinquency Distribution** — Bar chart of accounts by DPD bucket
3. **Vintage Curves** — Line chart of delinquency rates by origination month and risk tier
4. **Risk Scatter** — Bubble chart of credit score vs delinquency by tier
5. **ML Risk Model** — Logistic regression predicting delinquency probability by credit score
6. **AI Summary** — Snowflake Cortex generates an executive summary from the data
7. **PowerPoint Export** — 7-slide deck with native editable charts (not images)

## Data Requirements

The app expects 3 tables in `{DATABASE}.{SCHEMA}`:

| Table | Rows | Description |
|-------|------|-------------|
| `PORTFOLIO_RISK_SUMMARY` | 1 | Aggregate portfolio metrics |
| `COLLECTIONS_EFFECTIVENESS` | 5+ | Delinquency bucket breakdown |
| `LOAN_PERFORMANCE_CUBE` | 50+ | Monthly performance by risk tier |

See `setup_mock_data.sql` for the exact schema and sample data.

## Connecting Your Own Data

1. Create or identify tables matching the schema in `setup_mock_data.sql`
2. Update `config.py` with your database, schema, warehouse, and role
3. Update `snowflake.yml` if deploying to Snowflake (database, warehouse, app name)

## PowerPoint Output

The exported PPTX contains native OOXML chart objects — fully editable in PowerPoint and Google Slides. Charts are not images. The deck is saved to `/tmp/` and uploaded to the configured Snowflake stage.

- **Streamlit:** Click "Generate PowerPoint" then "Download Presentation"
- **Notebook:** Run all cells; the PPTX is saved to `/tmp/portfolio_analysis.pptx` and PUT to your configured stage

## Cortex Code Skill: `snowflake-data-to-pptx`

This repo includes a **Cortex Code skill** in `skills/snowflake-data-to-pptx/` that teaches the agent how to generate native-chart PowerPoint decks from *any* Snowflake dataset — not hardcoded to the loan portfolio schema used in this project.

### What the skill does

When a user asks Cortex Code to "create a PowerPoint from my Snowflake data" or "export to pptx", the skill provides a 5-step workflow:

1. **Discover** — Query tables, classify columns (categorical, numeric, date)
2. **Plan** — Propose slide structure (title, KPIs, charts, AI summary) and get approval
3. **Generate** — Build native python-pptx charts using the recipes in `references/`
4. **Persist** — Save to `/tmp/`, PUT to stage, generate presigned download URL
5. **Cortex AI** — Optionally add an AI-generated executive summary slide

### Installing the skill

Copy the skill directory to your Cortex Code skills location:

```bash
cp -r skills/snowflake-data-to-pptx ~/.snowflake/cortex/skills/
```

The skill will be auto-discovered by Cortex Code on next launch via trigger phrases like:
`pptx from data`, `powerpoint from snowflake`, `export to pptx`, `native chart`, `editable chart`

### Reference files

## Known Limitations

- The PPTX generation code is duplicated between the Streamlit app and notebook (not shared as a module). Minor drift exists in column names and return signatures, but both produce identical output.
- The ML model is a simple logistic regression for demonstration purposes — not suitable for production risk scoring.
- Cortex AI summary quality depends on the model and data volume. Large datasets should be pre-aggregated before sending to the LLM.
